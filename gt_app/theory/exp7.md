In graph theory, the shortest path problem involves finding a path between two vertices in a graph such that the total sum of edge weights along the path is minimized. This problem appears constantly in real-world applications such as GPS navigation systems finding the fastest route between two locations, network routing protocols determining the most efficient path for data packets, airline systems computing the cheapest flight connections, and logistics companies optimizing delivery routes. Among the various algorithms developed to solve this problem, Dijkstra's algorithm stands out as the most widely known and practically used solution for graphs with non-negative edge weights.

Given a weighted graph where each edge carries a non-negative numerical weight, the shortest path between a source vertex and a destination vertex is the sequence of edges whose cumulative weight is smallest among all possible paths connecting those two vertices. It is important to note that the shortest path does not necessarily mean the path with the fewest edges. A path with more edges but smaller weights can easily be shorter than a direct path with a single heavy edge. The concept of shortest path extends beyond just two vertices. A single-source shortest path problem asks for the shortest path from one source vertex to all other vertices in the graph, which is exactly what Dijkstra's algorithm solves.

Dijkstra's algorithm was conceived by Dutch computer scientist Edsger W. Dijkstra in 1956 and published in 1959. It is a greedy algorithm that solves the single-source shortest path problem for a graph with non-negative edge weights. The algorithm maintains a set of vertices whose shortest distance from the source has already been finalized, and it repeatedly selects the unvisited vertex with the smallest known distance, processes it, and updates the distances of its neighboring vertices. This process continues until the shortest distances to all vertices have been determined.

The greedy intuition behind Dijkstra's algorithm is based on a simple observation. If you always process the vertex that currently has the smallest tentative distance from the source, you can be certain that this distance is final and cannot be improved later. This is because all edge weights are non-negative, meaning no future path through other vertices can reduce the already-minimum distance. This greedy choice at each step, always picking the closest unvisited vertex, guarantees that the algorithm produces correct shortest paths without ever needing to revise a finalized distance.
